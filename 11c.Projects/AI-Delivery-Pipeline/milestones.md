# Milestones — AI-Delivery-Pipeline

The project is a long-running research hub, not a closeable project. The milestones below are
forcing functions to keep the work grounded — retro each one as it lands; the project itself stays
open.

## M1 — Near-term artifact (current focus)
Produce an AIX-handbook-style section — *"AI delivery pipeline"* — distilled into permanent notes in
`11l.LtS/`. Forcing function: the AIX-offering explanation grilling sessions (see Deferred
`aix-service-offering-explanation-plan` — 3 sessions: big picture/vision/intent, system design,
the actual software app).

When M1 lands: run `project-retro` on the artifact milestone, then continue.

## M2 — Reversibility / off-ramp (after M1)
Formalize fallback for when the AI layer degrades — model regression, provider outage, cost spike.
Pipelines that assume AI as a single point of failure are the threat to address.

## M3 — Tooling diversification
Move off front-running-only solutions toward provider-agnostic options, as the industry matures.
Pragmatic, not immediate — use front-running for enterprise now, diversify later.

## M4 — Baseline-measurement
Define observability + delivery-dashboard metrics for the "raising the baseline" outcome, in
consultation with other departments. Confirms the side-effect claim is real, not just asserted.
