# User flows

## 1. AI-assisted onboarding flow

CDI admin adds a new end user (company) to the system. Must gather all relevant company information as AI context:
- Regular billings
- Assets
- Categories
- Any other context the AI needs

## 2. Chat upload flow (main end user)

- Chat UI interface for the reporter/declarant
- Maximum ease of use — infer as much as possible from context
- User should not need to adjust data much after upload
- Workflow: upload bill → AI reads + enriches → user confirms

## 3. Walk-in demo / unknown business onboarding

- A person from a company visits the site, uploads a single bill
- System produces a quick ESG representation
- Used for C-level demos during CDI's sales meetings
- "Try before you buy" flow
