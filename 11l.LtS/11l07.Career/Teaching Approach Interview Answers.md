# Teaching Approach Interview Answers

Teaching approach answers explain technical work as if helping another engineer understand the reasoning.

Shape:

1. Start from the need, not the tool.
2. Define the concept in plain language.
3. Explain why the tool or pattern fit.
4. Mention tradeoffs and limits.
5. Check that the interviewer is following.

Example frame: “We needed asynchronous processing because synchronous requests were timing out. Kafka let us decouple ingestion from processing.”

This is useful when the interviewer is assessing communication, mentoring, cross-functional work, or unfamiliar technology.
